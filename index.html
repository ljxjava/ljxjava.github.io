【想法】现代自学心得

发表于 2019-07-21 | 更新于 2019-07-22 | 分类于 想法

以下文章仅为个人见解，请各位自行取舍

再次声明以下，我目前并不是【程序员】，不要总是认为会点电脑的就是程序员，计算机领域也不止程序员与黑客。

我只是一个喜欢了解事物原理，对计算机（等复杂系统）极其衍生领域感兴趣的人，从芯片制造，电路设计，计算机组成，嵌入式，到编译原理，操作系统，计算机网络，再到编程，软件使用，游戏开发，计算机图形学，机器学习…这些都是我想知道的。也正是因为我想知道的太多，得找个突破口来入门，于是就决定先从【编程】（对【计算机】抽象后的中间层）开始，逐步向两头的领域扩展学习。而在学校中只有编程能力有机会显现出来，才造成了这样的印象。（好歹我初中就设计过加法器电路，玩过unity）

方法篇

  • 你要明白什么？

    • 高中阶段自学高考内容以外的东西对大多数人来说没有什么用。
    • 国内好的自学教程较少
    • 通常需要看多个教程才能明白一个知识
    • 自学与在学校上课有很大不同，在学校学习大多数时候是被动的，而自学主动权在自己，可以自己选择学什么，向谁学。不是老师教你什么，而是你想学什么。
    • 自学本来就是依靠【兴趣】和【目标】的。自学就应该围绕【兴趣】和【目标】展开。
    • 【兴趣】能带你入门，了解它的整体结构。是搭框架的时候，还不是扣细枝末节的时候。
    • 【目标】能让你进阶，由框架向两级延申，向上学习它的高级技术应用，在应用中，向下深入它的底层技术原理。

  • 兴趣如何保持？

    • 成就感。这也是前期学习最重要的。

  • 如何获得成就感？

    • 认清认清现实，了解自己，降低自己一开始的目标期望，如果不到大学以及技术公司里学习，自学是极其难以达到该技术的前沿的，但是一定可以明白这些技术的原理。
    • 选择好的教程（视频，文章，书籍…）

  • 认清什么？

    • 知识有限，时间有限，选择好的教程
    • 目标有限，金钱有限，不要急于买书

  • 看哪些教程？

    • 不适合初学者的（信息密度太低或太高的，无效信息较多的，太枯燥的，太难的）

      视频类

      • 一上来就是直接说话，连稿子也不带的。信息密度太低，一直盯着人看很难受的，而且通常没有逻辑，难以让人明白什么。

      • 解说没有准备ppt，现场手绘的。节奏较慢，结构也不清晰，难以让人明白动态的过程，看着也很难受。

      • 直播形式的各类培训网课。这类视频通常节奏很慢，废话很多，内容也通常只涉及表皮。

      • 大学专业课教室录播。信息密度太高，讲解的时候附带了很多初学者没有必要了解的知识，反而消磨了初学者的兴趣和时间，比较枯燥，前期难以获得成就感。

        大学专业课程很大程度上依赖与大学数学物理，直接看专业课程肯定会有些看不懂。他们的课程本来就是为这些人设计的，很多时候没有考虑你的层次。

      文章类

      • 各类文库里的（百度，360）。里面收集的大都是论文，很难看懂，有大量重复文章。
      • CSDN里的。大量抄袭转载的文章，大都比较老旧，粗略看看就行了。

      书籍类

      • 大学教材。同视频类，适合有老师带着学，大多数不适合自学。
      • 专业性很强的书籍（XX导论，XX原理，XX技术…..）书很重，很厚，不方便翻阅携带，通常很枯燥，前期难以获得成就感，语言很平淡，很容易劝退初学者，而且要么只有理论，要么只有实践，要么是理论和实践都没有讲好。
      • 明日科技的XX从入门到精通系列。内容软件大都很老， 比喻不当，毁人不倦。

    • 适合初学者的

      视频类

      • 慕课里的（中国大学MOOC）。里面的视频不同于教室教室录播，对自学者要更友好些，有配合ppt讲解，比手绘

        更容易让人理解。

      • b站里的。有有很多搬运工搬运外站的教程，如果不是英语非常好，建议还是看翻译过的，机翻的也可以看看，以及个别up主的视频。

      文章类

      • 知乎里的。在问题里找高赞的回答。

      书籍类

      • 销量高的。虽然简单粗暴，但是也是个保底的方法，至少不会买到垃圾书。
      • 语言幽默风趣的（例如大话XX）。别不乐意，觉着技术含量低，但是他们去除了部分初学者确实不需要掌握的两极知识，能用幽默的语言让你明白这个技术的结构框架，看着有劲，有成就感，通常也有许多实践，适合初学者。

  • 怎样看教程？学习顺序？

    • 建议初学者想学什么都在mooc里搜一下，看看播放量高的视频就行了。粗略地地过一遍，因为很多时候，面对一个自己陌生的领域，你甚至不知道这个领域到底要学什么。看慕课可以让你清楚地认识到，什么是我不明白的，什么是我想知道的，有了这样的【目标】，你才能选择自己想看的教程，才能知道哪些书是对自己有用的。
    • 买书前都先去去网上搜一搜资源（脚本之家），细致地翻一下。虽然看盗版不太好，但也避免了自己买到不想要的书。（165买个教训）
    • 有选择性的学习。有的书会给出知识之间的依赖关系，可以选择自己感兴趣的部分学习，主动滤除掉不需要了解的知识，能节约时间，更块获得成就感
    • 遇到不懂地方不要死磕。好好利用网络搜索你的问题，而不是反复看当前教程，停留在那，这只会逐渐消耗你的兴趣。要习惯于看多个教程，这个看不懂了就去看另一个，从多个角度看能更容易理解。如果还是找不到能看懂的教程就跳过这部分，去看下一部分，也许看来后面再来看前面反而理解了。

【想法】书籍写作

发表于 2019-06-13 | 分类于 想法
  • 如何写好一本入门书，读者友好型书，也是一门学问。
  • 背景
    • 很多书籍说的”快速入门“”适合初学者“，可还是有很多人看不懂，实际内容还是会让小白懵逼。
  • 分析问题
    • 首先是对读者的情况不明确，只按照自己的思路写下去，不曾想到那些自认为很简单，常识的内容，背后有着各方面的大量的知识积累。
    • 很多专业技术强的人不知道如何表述
    • 许多教材的缺点是重理论轻实践，与实际工作生产脱节，常常是前几章全在讲理论，而不讲这些理论知识对接下来的学习有什么用，学完一个知识点不知道有什么用，导致前期无聊，读者没有成就感；而后面的章节常常直接将高级技术，没有过渡，一上来就要把所有问题，最优解告诉你，却不说明这样做的原因。
    • 许多教材所用的软件工具老旧
      • 对于初学者，vscode是最好的选择，也够用了。vscode作为一个现代软件，至少UI界面简洁美观，符合现代审美；轻便，开源免费；使用人数多，社区活跃；插件多，教材多。
  • 解决问题
    • 最重要的要让读者知道为什么要这样设计而不那样设计，否则初学者心中始终有疑惑，继续学就会很难受
    • 学到每一点知识都能举例应用，既有成就感，又加深了记忆
    • 多用打比方，举例子，但比喻举例一定要恰当
    • 考虑到目前碎片化阅读的情况越来越普遍，

【想法】语法统一

发表于 2019-06-07 | 分类于 想法
  • 背景
    • 目前众多的编程语言语法各有差异，经常容易记混。
    • 通常语言的语法并非语言之间最本质的区别，却增加了学习负担。
    • 喜欢一个语言的设计理念，却不喜欢它的语法（比如说rust，真的是一种浑身难受的感觉）。
  • 概念
    • 开发一个软件，可以自定义语法，相当于从中间语言转换为目标语言。
    • 拥有图形化编程功能，文字与图形交叉，类似实时渲染的Markdown.
    • 高度可定制化的UI界面，参考solidworks

【想法】标签化网络收藏整理软件

发表于 2019-06-06 | 更新于 2019-06-07 | 分类于 想法
  • 背景
    • 获取知识后无意识整理。例如看视频的时候常常是“投币点赞收藏退出出一气呵成”，总是趋向于获得更多的信息，却又不去花时间整理（心有余而力不足的感觉），没有一个学习任务清单。
    • 基于目前各大网络平台的自带的收藏系统难以高效整理。
    • 各网络平台都有自己的收藏系统，各有优劣，导致收藏内容过于分散，不便查找，相互之间不能导入导出。
    • 常常会下载一堆乱七八糟的文件也不去分类整理，最后干脆就一次性删除了。
  • 概念
    • 开发一个第三方软件，用于收集汇总并整理在各平台的收藏内容，参考必收，为知。
    • 能把各个平台收藏时的操作统一，收藏时临时界面应快速弹出响应，操作简单流畅。
    • 能导入各个平台的收藏夹，类似于listen1可以从各个平台导入歌单。
    • 能导出到本地，避免意外情况下的收藏丢失。
    • 即使有时忘记使用此第三方软件来收藏，也能自动爬取下你在各平台收藏夹的更新，自动添加到待整理任务列表，每日提醒，打卡任务（任务驱动）。
    • 不仅能整理网络收藏，也能整理本地文件。选择一些文件夹，自动检测下载的新文件，自动添加到待整理任务列表。
